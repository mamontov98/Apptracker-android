# SDK Configuration

## Automatic Initialization

The SDK is automatically initialized when the app starts, in `AppTrackerApplication.kt`.

**Important**: The SDK automatically creates a project in the backend if it doesn't exist. You don't need to create it manually!

## How to Configure

### Option 1: Edit AppTrackerApplication.kt (Recommended for Development)

1. Open `android/app/src/main/java/com/apptracker/demo/AppTrackerApplication.kt`
2. Change the default values in `getProjectName()` and `getBaseUrl()`:

```kotlin
private fun getProjectName(): String {
    // Change this line to your project name:
    return "Your App Name"
}

private fun getBaseUrl(): String {
    // Change this line:
    return "http://10.0.2.2:5000" // For emulator
    // or
    return "http://192.168.1.100:5000" // For physical device (use your computer's IP)
    // or
    return "https://your-api-domain.com" // For production
}
```

**Note**: The project key is automatically generated by the backend and saved for future use.

### Option 2: Set Programmatically (For Runtime Configuration)

You can set the project key and base URL programmatically from anywhere in your app:

```kotlin
val prefs = getSharedPreferences("apptracker_config", Context.MODE_PRIVATE)
prefs.edit()
    .putString("project_key", "your-project-key")
    .putString("base_url", "http://10.0.2.2:5000")
    .apply()

// Then restart the app or reinitialize SDK
```

### Option 3: Use Build Config (For Production)

1. Add to `android/app/build.gradle.kts`:
```kotlin
android {
    defaultConfig {
        buildConfigField("String", "APP_TRACKER_PROJECT_KEY", "\"your-project-key\"")
        buildConfigField("String", "APP_TRACKER_BASE_URL", "\"https://your-api.com\"")
    }
}
```

2. Use in `AppTrackerApplication.kt`:
```kotlin
return BuildConfig.APP_TRACKER_PROJECT_KEY
return BuildConfig.APP_TRACKER_BASE_URL
```

## Current Default Values

- **Project Name**: `Android Demo App` (project key is auto-generated)
- **Base URL**: `http://10.0.2.2:5000` (for emulator)

**Important**: Change these values before using in production!

## How It Works

1. When the app starts, `AppTrackerApplication` runs
2. It checks if a project key is saved in SharedPreferences
3. If saved, it verifies the project still exists in the backend
4. If not saved or doesn't exist, it creates a new project with the project name
5. The generated project key is saved for future use
6. The SDK is initialized with the project key

## Testing

1. Make sure your backend is running
2. Create a project in the backend with the project key you're using
3. The SDK will automatically start tracking events when the app launches
4. Check events in the backend dashboard or Swagger UI
